## `time.sh` 说明
这个脚本用于精确控制文件或目录的访问时间（atime）、修改时间（mtime）甚至创建时间（crtime）。
*   **软件编译与构建**：像 `make` 这样的构建工具依赖文件的修改时间来判断哪些源文件需要重新编译。当你从 Git 等版本控制系统克隆或切换分支后，文件的修改时间会变成当前时间，可能导致不必要的全量编译。你可以使用此脚本将文件时间恢复到某个特定节点，从而实现增量编译。
*   **测试自动化**：在测试需要处理旧文件或日志的程序时（例如，测试日志轮转、临时文件清理功能），你不再需要手动 `sleep` 或等待。使用此脚本，可以瞬间制造出“N天前”的文件，极大地提高了测试效率。
*   **数据同步与备份**：某些备份或同步工具是基于文件时间戳来决定是否需要同步的。如果你想强制备份某些文件，或者在迁移数据后希望保持原始时间戳，这个脚本就能派上用场。
*   **文件整理与归档**：在整理照片、文档等个人文件时，有时文件的元数据（时间戳）可能因为复制、编辑而丢失。你可以用这个脚本将文件的时间戳批量修改为某个有意义的时间，方便后续按时间排序和查找。


## 如何使用 `time.sh`

这个脚本无需修改任何内部代码，所有操作都通过命令行参数完成。

**1. 保存脚本**
将上面的代码保存为 `time.sh` 文件。

**2. 给予执行权限**
```bash
chmod +x time.sh
```

**3. 使用示例**

*   **场景一：将文件 `test.log` 的时间修改为当前时间。**
    ```bash
    ./time.sh ./test.log
    ```

*   **场景二：想测试一个处理旧文件的程序，将 `data.txt` 的时间修改为2023年元旦。**
    ```bash
    ./time.sh ./data.txt '2023-01-01 08:30:00'
    ```

*   **场景三：刚 `git clone` 了一个项目，想避免全量编译，将整个项目目录 `my-project` 的所有文件和子目录时间都改掉。**
    ```bash
    # -r 表示递归 (recursive)
    ./time.sh -r ./my-project '2025-01-15 10:00:00'
    ```

*   **场景四：只想更新文件的访问时间（atime），而不改变修改时间（mtime）。**
    ```bash
    # -a 表示只修改访问时间 (access)
    ./time.sh -a ./important-file.conf
    ```

*   **场景五：强制执行，跳过烦人的确认步骤（请谨慎使用！）。**
    ```bash
    # -f 表示强制 (force)
    ./time.sh -r -f ./logs '2024-05-01 00:00:00'
    ```
